Mise en place initiale

Une fois le module installé, vous devez configurer le système pour qu’il exécute le script cronical périodiquement. Ce script vérifiera si des actions ont besoin d’être exécutés et les exécutera.

Il y a plusieurs façons de faire ça. Nous allons expliquer les plus communes.

Notez que dans les exemples qui suivent nous configurons le système pour lancer le script cronical chaque minute. Ce n’est pas obligatoire. Par exemple vous pouvez le lancer une fois par heure et il exécutera toutes les actions qui auraient dû être exécutées durant l’heure précédente. Donc ajustez selon vos besoins.

cron

Créez /etc/cron.d/cronical avec le contenu suivant:

* * * * * <user> <path-to-omeka-s>/modules/Cronical/bin/cronical

Remplacez <user> par le nom de l’utilisateur qui exécute Omeka S (www-data par exemple), et remplacez <path-to-omeka-s> par le chemin absolu d’Omeka S.

Si vous n’avez pas d’accès root, mais que vous pouvez vous connecter avec le même utilisateur que celui qui exécute Omeka S, vous pouvez exécuter crontab -e et ajouter la ligne suivante:

* * * * * <path-to-omeka-s>/modules/Cronical/bin/cronical

systemd

Créez /etc/systemd/system/cronical.timer avec le contenu suivant:

[Unit]
Description=Cronical

[Timer]
OnCalendar=*-*-* *:*:00

[Install]
WantedBy=timers.target

et créez /etc/systemd/system/cronical.service avec le contenu suivant:

[Unit]
Description=Cronical

[Service]
Type=oneshot
ExecStart=<path-to-omeka>/modules/Cronical/bin/cronical
User=<user>

Remplacez <user> par le nom de l’utilisateur qui exécute Omeka S (www-data par exemple), et remplacez <path-to-omeka-s> par le chemin absolu d’Omeka S.

Puis exécutez la commande suivante:

systemctl enable --now cronical.timer
